# teambuilder

A repository exploring how to use Claude Code to implement a team-building application. The core problem: take a set of participants registered for an event and divide them into teams using random or ability-balanced strategies.

Each subdirectory contains a separate implementation, generated using a different Claude Code workflow or tooling approach.

## Implementations

| Directory | Language | Tool | Description |
|---|---|---|---|
| [teambuilder-sdd](teambuilder-sdd/) | Java | Claude Code | Spec-driven development using Claude Code skills |
| [teambuilder-attractor](teambuilder-attractor/) | JavaScript | [Attractor](https://factory.strongdm.ai/products/attractor) | Generated by StrongDM's non-interactive coding agent |
| [teambuilder-superpowers](teambuilder-superpowers/) | JavaScript | [Superpowers](https://github.com/obra/superpowers) | Generated using Jesse Vincent's structured agentic workflow |

## The Problem

Given a list of participants (each with a name and ability score), build teams subject to configurable size constraints. Two formation strategies are supported across all implementations:

- **Random** — participants are shuffled and assigned sequentially
- **Fair** — participants are sorted by ability score and distributed using a snake-draft pattern, balancing team skill sums

All implementations handle edge cases gracefully (empty input, invalid size constraints, participant overflow) without throwing exceptions.
