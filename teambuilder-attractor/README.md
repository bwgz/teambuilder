# teambuilder-attractor

A JavaScript library for distributing participants into balanced teams using RANDOM or FAIR (snake-draft) strategies.

## Usage

```js
import {
  createParticipant,
  createTeamConfig,
  createTeamBuilder,
  Mode,
} from './src/index.js';

const participants = [
  createParticipant({ id: '1', name: 'Alice', ability: 9.0 }),
  createParticipant({ id: '2', name: 'Bob',   ability: 7.5 }),
  createParticipant({ id: '3', name: 'Carol', ability: 8.0 }),
  createParticipant({ id: '4', name: 'Dave',  ability: 6.0 }),
];

const config = createTeamConfig({ idealSize: 2, minSize: 2, maxSize: 2, mode: Mode.FAIR });

const builder = createTeamBuilder();
const result  = builder.buildTeams(participants, config);

console.log(result.success);  // true
console.log(result.message);  // "Teams formed successfully"
result.teams.forEach((team, i) => {
  console.log(`Team ${i + 1}:`, team.getParticipants().map(p => p.name));
});
```

## API

### `createParticipant({ id, name, ability })`

Creates an immutable participant value object.

| Field     | Type   | Description                  |
|-----------|--------|------------------------------|
| `id`      | String | Unique identifier            |
| `name`    | String | Display name                 |
| `ability` | Float  | Skill score (any numeric value) |

### `createTeamConfig({ idealSize, minSize, maxSize, maxTeams?, mode })`

Creates an immutable team configuration.

| Field       | Type    | Description                              |
|-------------|---------|------------------------------------------|
| `idealSize` | Integer | Preferred team size                      |
| `minSize`   | Integer | Minimum acceptable team size             |
| `maxSize`   | Integer | Maximum acceptable team size             |
| `maxTeams`  | Integer | Hard cap on teams (default/max: 100)     |
| `mode`      | Mode    | `Mode.RANDOM` or `Mode.FAIR`             |

### `createTeamBuilder()` / `new TeamBuilder()`

Returns a builder instance with a single method:

#### `buildTeams(participants, config) -> TeamResult`

| Field       | Type               | Description                              |
|-------------|--------------------|------------------------------------------|
| `success`   | Boolean            | `true` if at least one team was formed   |
| `teams`     | `Team[]`           | Formed teams                             |
| `unplaced`  | `Participant[]`    | Participants who could not be placed     |
| `message`   | String             | Human-readable status message            |

### `Mode`

```js
Mode.RANDOM  // Shuffle participants randomly before distributing
Mode.FAIR    // Snake-draft by ability score for balanced teams
```

## Strategies

- **RANDOM** — shuffles participants and assigns them sequentially to teams.
- **FAIR** — sorts participants by ability descending, then distributes using a snake-draft pattern (left-to-right, then right-to-left) to balance ability sums across teams.
